Como correr o cleanup (dry-run e real)

Dry-run (ver o que aconteceria):

python3 scripts/cleanup_optimize.py --root projects --min-csv-mb 10 --dry-run

Execução real:

python3 scripts/cleanup_optimize.py --root projects --min-csv-mb 10

O script faz backup automático dos ficheiros que modifica dentro de --root (pasta cleanup_backups_YYYYMMDD_HHMMSS).

Restaurar apenas alguns ambientes
  1. Edite projects/venv_removal_backup_20260515_220444/venv_to_remove_ordered.txt e mantenha só as linhas dos ambientes que quer recriar.
  2. Chame o script com o mesmo backup dir (o script lê esse ficheiro):
bash scripts/recreate_envs.sh /Users/carlosferreira/Projecto/projects/venv_removal_backup_20260515_220444
Dicas e notas de segurança
  • Antes de eliminar ficheiros, verifique os backups criados em projects/cleanup_backups_* e projects/venv_removal_backup_*.
  • Conversão para Parquet usa pyarrow. Se não estiver instalado, o script fará fallback para gzip. Para instalar: pip install pyarrow.
  • Os ambientes virtuais ocupavam vários GB — o script de recriação instala pacotes conforme os ficheiros de requisitos, pode demorar.
  • Se preferir usar poetry ou pipenv, diga-me para adaptar scripts/recreate_envs.sh.
Se algo correr mal
  1. Verifique os backups em projects/cleanup_backups_* — os ficheiros originais foram copiados antes de serem removidos.
  2. Se um venv falhar ao instalar, execute manualmente dentro do venv: source <venv_path>/bin/activate && pip install -r requirements.txt
  3. Peça-me para adaptar o processo (ex.: restaurar só DBs, mover ficheiros para cloud, ou reverter uma execução específica).

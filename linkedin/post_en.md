# LinkedIn Post - English

I built a central portfolio hub for my Data Engineering, Data Science and Analytics projects:

**Projects_data_eng_science_analyst**

GitHub:
https://github.com/JoseCCFerreira/Projects_data_eng_science_analyst

This repository is not just a collection of notebooks. It is a central access layer that connects multiple end-to-end projects with:

- Streamlit dashboards
- SQL and relational modeling
- SQLite and DuckDB databases
- dbt-style analytics engineering
- data pipelines
- statistical analysis
- machine learning
- visual documentation
- bilingual English/Portuguese explanations
- operational scripts to start and test every dashboard

The central dashboard gives access to six projects:

1. Global Fuel Shocks Intelligence
2. Citibike E-Bike Performance
3. Portugal Bike Geospatial ML
4. Retail Analytics Learning
5. Retail Case With Beginner HTML
6. Tyre Wear Intelligence

Each project is independently runnable, but the central repository acts as the portfolio control room: dashboards, documentation, repository links, databases, data files, charts, Git status, service logs and project explanations are all accessible from one place.

What I wanted to demonstrate:

- how raw data becomes analytical outputs
- how relational and analytical databases fit together
- how SQL, Python, dbt, statistics and ML connect in real workflows
- how dashboards can make technical projects easier to explore
- how documentation can turn a codebase into something understandable

The repository includes diagrams explaining:

- central architecture
- project portfolio
- data pipeline
- Git workflow
- mathematics/statistics/ML concepts
- dashboard access map

This was built as a practical portfolio system for data analyst, data engineer, analytics engineer and data science roles.

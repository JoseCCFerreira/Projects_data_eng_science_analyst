- Otimizado para OLAP (consultas em lote)
- Integrado com dbt

✅ ETAPA 4: QUALITY-REPORT
Origem: src/quality/data_quality_report.py
Saída: reports/data_quality_report.md
- Análise: valores nulos, duplicados, outliers (IQR), correlações
- Estatísticas: mean, median, std, skew, kurtosis
- Recomendações: normalizar, validar ranges, isolar anomalias

✅ ETAPA 5: CLEAN-DATA
Origem: src/preprocessing/cleaning.py
Saída: data/processed/clean_sensor_readings.csv
- Entrada: 32,130 linhas
- Saída: 31,675 linhas (455 duplicados/nulos removidos)
- Ações: normalizar categorias, validar ranges físicos, preencher valores nulos
- Redução: 1.4% de dados removidos

✅ ETAPA 6: RESAMPLE
Origem: src/preprocessing/resampling.py
Saída: data/processed/resampled_hourly.csv, resampled_daily.csv
- Resampling: groupby(hora/dia, máquina) com mean, min, max, std, count
- Horário: agregações por sensor/máquina em períodos de 1 hora
- Diário: agregações em períodos de 1 dia
- Reutilizável para análises temporais

✅ ETAPA 7: FILTER-NOISE
Origem: src/preprocessing/noise_filtering.py
Saída: data/processed/filtered_sensor_readings.csv
- Técnicas: rolling average (window=5), moving median, z-score (threshold=3), Savitzky-Golay
- Mantém variabilidade natural enquanto remove anomalias
- Preparado para ML

✅ ETAPA 8: TRAIN-CLUSTERING
Origem: src/ml/clustering.py
Saída: data/exports/clustering_results.csv, models/clustering/
- 3 modelos treinados:
  → K-Means (4 clusters): agrupa por similaridade global
  → DBSCAN: identifica densidade e ruído
  → Agglomerative: hierarquia de clusters
- Features: temperatura, humidade, pressão, vibração, energia, risco
- PCA 2D para visualização de 6D → 2D
- Scaler, PCA e modelos salvos como .joblib

✅ ETAPA 9: TRAIN-PREDICTION
Origem: src/ml/prediction.py
Saída: data/exports/predictions.csv, models/predictive/
- Alvo: target_failure_risk (regressão)
- Train/test: 80/20 split
- Modelos:
  → Linear Regression: baseline
  → Random Forest: não-linear (100 árvores)
- Métricas: MAE, RMSE, R²
- Modelos salvos para deploy

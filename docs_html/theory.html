Mean, Variance and Standard Deviation

Formula: mean = sum(x_i)/n; variance = sum((x_i - mean)^2)/(n - 1); std = sqrt(variance).

Example: for [2, -1, 4, 0], the mean is 1.25. Standard deviation tells us whether values are tightly grouped or spread out.

Project use: fuel volatility, retail sales variation, bike usage distributions and tyre wear variability.

Z-Score and Outliers

Formula: z = (x - mean) / standard_deviation.

Example: z = 2.8 means the value is 2.8 standard deviations above typical behavior.

Project use: fuel price shock detection, unusual sales periods, mobility anomalies and tyre wear alerts.

Correlation

Formula: corr(X,Y) = cov(X,Y)/(std(X)*std(Y)).

Example: if two series rise together, correlation tends to be positive. It does not prove causality.

Project use: price/event relationships, sales drivers, bike demand factors and tyre wear associations.

Time-Series Returns

Formula: return_t = (value_t / value_{t-1} - 1) * 100.

Example: if price moves from 100 to 110, return is 10%. If it moves from 110 to 99, return is -10%.

Project use: fuel monthly changes, sales evolution, demand shifts and tread-depth change.

Regression

Formula: y = beta_0 + beta_1*x_1 + ... + error.

Example: predict sales from price, discount and seasonality. beta_1 describes the marginal change associated with x_1.

Project use: forecasting baselines, interpretable models and comparison against more flexible machine-learning methods.

Classification Metrics

Formula: accuracy = correct/total; precision = TP/(TP+FP); recall = TP/(TP+FN).

Example: in a price-shock model, a false positive is a shock warning without a real shock; a false negative misses a real shock.

Project use: risk flags, price jumps, recommendations and decision-support classifiers.

Clustering

Formula: Euclidean distance = sqrt(sum((x_i - y_i)^2)).

Example: stores can be grouped by sales, margin and customer count; tyres can be grouped by wear behavior.

Project use: segmentation in retail, mobility, tyre wear and operational behavior.

Machine-Learning Workflow

Flow: define target, create features, split train/test, train baseline, train model, evaluate, interpret, document limitations.

Example: a Random Forest uses many decision trees to model nonlinear relationships and returns feature importance values.

Project use: fuel jump risk, retail prediction, geospatial prioritization, bike demand patterns and tyre recommendations.
